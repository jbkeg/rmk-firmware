# Security Policy

## Supported Versions

This repository is maintained as a rolling RMK firmware project.

| Version / Branch | Supported | Notes |
| --- | --- | --- |
| `main` | :white_check_mark: | Primary maintenance branch. |
| Latest release tag | :white_check_mark: | Security fixes may be backported when practical. |
| Older tags | :x: | Upgrade to `main` or latest release. |
| Forks/archives | :x: | Not maintained by this repository. |

## Reporting a Vulnerability

Please report vulnerabilities privately through GitHub Security Advisories:

1. Open the repository `Security` tab.
2. Select `Report a vulnerability`.
3. Provide impact and reproduction details.

Do not open public issues for unpatched vulnerabilities.

## Include in Report

- Affected target artifacts and profiles
- Commit SHA, branch, or release tag
- Reproduction steps
- Expected impact and attack scenario
- Suggested mitigation (if known)

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage decision: within 7 business days
- Status updates: at least weekly while active
